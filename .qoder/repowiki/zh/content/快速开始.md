## 简介
本指南面向首次接触 OpenClaw 的用户，目标是在最短时间内完成安装、运行并体验 AI 助手的核心能力。你将通过交互式向导完成网关安装、工作空间与技能配置，并可直接在浏览器中进行首次对话。

- 推荐路径：使用安装脚本一键安装并启动向导；或直接运行 `openclaw onboard` 进入交互式设置。
- 支持平台：macOS、Linux、Windows（WSL2 强烈推荐）。
- 运行时：Node ≥22。

## 安装方式与选择建议
以下提供多种安装路径，按“推荐优先级”排序：

- 方式一：安装脚本（最简路径）
  - macOS/Linux/WSL2：`curl -fsSL https://openclaw.ai/install.sh | bash`
  - Windows（PowerShell）：`iwr -useb https://openclaw.ai/install.ps1 | iex`
  - 可跳过向导仅安装二进制：追加参数 `--no-onboard`（对应平台）

- 方式二：npm / pnpm
  - npm：`npm install -g openclaw@latest` 后执行 `openclaw onboard --install-daemon`
  - pnpm：`pnpm add -g openclaw@latest` 后执行 `pnpm approve-builds -g`（出现构建脚本警告时），再运行 `openclaw onboard --install-daemon`

- 方式三：从源码构建（贡献者/开发者）
  - 克隆仓库后：`pnpm install` → `pnpm ui:build` → `pnpm build`，然后运行 `openclaw onboard --install-daemon`

- 方式四：Bun（实验性）
  - 适合本地开发与热重载，但不建议用于生产网关（部分通道存在兼容性问题）。如遇生命周期脚本被阻止，可按文档指引信任相关包。

- 方式五：其他安装方式
  - Docker、Podman、Nix、Ansible、Bun CLI 等详见安装页卡片入口。

## 首次运行：控制台与向导
- 安装完成后，推荐运行交互式向导以完成网关、认证、通道与技能的配置：
  - `openclaw onboard --install-daemon`
- 若仅想快速体验聊天，无需通道配置，可直接打开控制面板：
  - `openclaw dashboard`（浏览器访问本地 127.0.0.1:18789）
- 如需前台运行网关进行测试或排障：
  - `openclaw gateway --port 18789`

提示：向导支持“快速开始”和“高级”两种模式；若需要非交互式自动化，可参考 CLI onboarding 文档中的非交互选项。

## 基础配置与常用任务
- 最小化配置示例（JSON5 支持注释与尾随逗号）：
  - 在 `~/.openclaw/openclaw.json` 中添加工作空间与允许的 DM 来源等字段
- 编辑配置的方式：
  - 交互式向导：`openclaw onboard` / `openclaw configure`
  - CLI 一次性命令：`openclaw config get/set/unset`
  - 控制面板：打开本地 127.0.0.1:18789 的“配置”标签页
  - 直接编辑文件：Gateway 会监听变更并热加载
- 常见任务：
  - 设置通道（WhatsApp/Telegram/Discord/Slack/Signal/iMessage/Google Chat/Mattermost/Teams 等）
  - 选择与配置模型（主模型与回退模型）
  - 控制 DM 访问策略（配对/白名单/开放/禁用）
  - 群组提及门控（正则模式）
  - 会话与重置策略（每日重置、空闲重置、线程绑定）

注意：严格校验模式下，未知键、类型错误或非法值会导致网关拒绝启动；可通过 `openclaw doctor` 查看并修复。

## 多平台安装要点
- macOS
  - 安装脚本会处理 Node 检测与安装；随后进入向导流程
  - 可在应用内安装全局 CLI 并配置守护进程
- Linux
  - 使用安装脚本或 npm/pnpm 安装；守护进程由 systemd（WSL2）或 LaunchAgent（原生 macOS）管理
- Windows（WSL2）
  - 强烈推荐通过 WSL2 安装与运行；安装脚本同样适用
  - 可通过 `openclaw onboard --install-daemon` 安装服务；必要时配置开机自启与端口转发

## 常见问题与排障
- 命令未找到（PATH 问题）
  - 检查全局前缀：`npm prefix -g`，并将该目录加入 PATH；重启终端后生效
- 权限错误（Linux）
  - 将 npm 全局前缀改为用户可写目录，并更新 PATH
- 网关无法启动/服务未运行
  - 使用诊断命令链：`openclaw status` → `openclaw gateway status` → `openclaw doctor` → `openclaw logs --follow`
  - 关注“未启用认证即绑定非回环地址”“端口占用”“服务未加载”等日志信号
- 控制面板或 Dashboard 无法连接
  - 校验网关 URL/端口、认证模式与令牌；确认 UI 正确指向本地或可达的网关地址
- 通道已连接但消息不流转
  - 检查提及门控、配对状态、权限令牌与允许列表
- 通道安装插件失败
  - 插件包需包含正确的 `openclaw.extensions` 字段并指向已构建的运行时文件
